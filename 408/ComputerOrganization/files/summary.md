## 核心矛盾图谱

### 一、计算机系统概述（🎨 涂料）

**矛盾**：程序是逻辑，机器是电路——怎么把"如果-那么"变成"通-断"？

**核心概念**：
- 冯·诺依曼结构：五大部件（控制器、运算器、存储器、输入设备、输出设备）
- 存储程序概念：指令和数据以二进制形式存储在存储器中
- 计算机体系结构（定义接口）vs 计算机组成（实现接口）

**跨科缝合**：⛓️ → OS-系统调用（用户态→内核态）

---

### 二、数据的表示与运算（🏗️ 梁柱）

**矛盾**：人的数学是无限精度的，机器寄存器是有限位宽的。

**核心概念**：
- **原码**：符号位 + 绝对值，直观但加减法复杂
- **补码**：负数用取反加一表示，加减法统一（最常用）
- **反码**：原码除符号位取反，中间过渡
- **定点数**：小数点位置固定
- **浮点数**：IEEE 754 标准（单精度 32 位/双精度 64 位）
- **ALU**：算术逻辑单元，执行加减乘除与或非等运算
- **移位运算**：算术移位（符号位不变）vs 逻辑移位（空位补 0）
- **溢出判断**：一位符号位法、双符号位法

**致命陷阱**：
| 陷阱 | 正确理解 |
|------|---------|
| 补码符号位参与运算 | 符号位参与二进制加法，溢出看最高位⊕次高位进位 |
| 浮点数精度 | 只有 2 的幂次组合能精确表示（0.1 + 0.2 ≠ 0.3） |
| 原码求补码 | 负数：除符号位取反加一 |

**跨科缝合**：⛓️ → 数据结构-位图（位运算应用）

### 三、存储器层次结构（🏗️ 梁柱）

**矛盾**：CPU 太快，内存太慢——速度差了几个数量级。

#### 主存储器

**核心概念**：
- **SRAM**（静态 RAM）：用触发器存储，速度快，集成度低，做 Cache
- **DRAM**（动态 RAM）：用电容存储，需刷新，集成度高，做主存
- **ROM**（只读存储器）：MROM → PROM → EPROM → EEPROM → Flash
- **存储器扩展**：位扩展（增加字长）、字扩展（增加容量）、字位扩展
- **海明码**：纠错编码，满足 2^r ≥ m + r + 1

**DRAM 刷新方式**：
| 方式 | 死区 | 特点 |
|------|:----:|------|
| 集中式 | 64μs | 死时间率 3.2% |
| 分散式 | 无 | 存取周期加倍 |
| 异步式 | 0.5μs | 将刷新安排在译码阶段 |

#### Cache 高速缓存

**核心概念**：
- **局部性原理**：时间局部性 + 空间局部性
- **地址映射**：直接映射、全相联映射、组相联映射
- **替换算法**：FIFO、LRU（近期最少使用）
- **写入策略**：写直达（写 Cache + 主存）、写回法（只写 Cache）
- **性能公式**：平均访问时间 = h × tc + (1 - h) × tm

**三种映射对比**：
| 映射方式 | 冲突率 | 硬件复杂度 | 灵活性 |
|----------|:------:|:----------:|:------:|
| 直接映射 | 高 | 低 | 差 |
| 全相联映射 | 低 | 高 | 好 |
| 组相联映射 | 中 | 中 | 中 |

**致命陷阱**：
| 陷阱 | 说明 |
|------|------|
| Cache 组号 vs 标记位 | 地址拆分时，组号位在中间，标记位在高位，块内地址在低位 |
| LRU 用堆栈实现 | 用"最近使用"的顺序栈，不是时间戳 |

**跨科缝合**：⛓️ → OS-页表（Cache 组相联 ↔ 多级页表映射）；⛓️ → OS-页面置换算法（LRU 在 Cache 和 OS 中的实现差异）

### 四、指令系统（🏗️ 梁柱）

**矛盾**：人的操作意图是"加两个数"，机器需要知道"用什么数、加到哪里、结果存哪"。

**核心概念**：
- **指令格式**：操作码 + 地址码
- **寻址方式**：立即/直接/隐含/间接/寄存器/寄存器间接/基址/变址/相对/堆栈
- **CISC vs RISC**：复杂指令集 vs 精简指令集

**寻址方式对比**：
| 寻址方式 | 有效地址 | 访问次数 | 用途 |
|----------|----------|:-------:|------|
| 立即寻址 | A 即为操作数 | 0 | 常数 |
| 直接寻址 | EA = A | 1 | 全局变量 |
| 间接寻址 | EA = (A) | 2 | 指针 |
| 寄存器寻址 | EA = Ri | 0 | 中间变量 |
| 基址寻址 | EA = (BR) + A | 1 | 重定位 |
| 变址寻址 | EA = (IX) + A | 1 | 数组遍历 |
| 相对寻址 | EA = (PC) + A | 1 | 程序转移 |

**CISC vs RISC**：
| 特性 | CISC | RISC |
|------|------|------|
| 指令条数 | 多 | 少 |
| 指令长度 | 可变 | 固定 |
| 通用寄存器 | 较少 | 多（32+） |
| 控制方式 | 微程序 | 硬布线 |
| 典型架构 | x86 | ARM、MIPS、RISC-V |

---

### 五、中央处理器（🏗️ 梁柱——大题常客）

**矛盾**：顺序执行太慢——每条指令都等上一条完成，CPU 利用率极低。

#### CPU 的结构

- **运算器**：ALU、ACC、MQ、X 寄存器
- **控制器**：CU、PC、IR
- **寄存器**：通用寄存器、数据寄存器、地址寄存器、条件码寄存器、PSW

#### 指令周期

- **取指周期**：PC → MAR → 主存 → MDR → IR
- **间址周期**：MDR 中形式地址 → MAR → 主存 → MDR（得到有效地址）
- **执行周期**：根据指令类型执行具体操作
- **中断周期**：保存断点 PC → 堆栈，取中断向量

#### 指令流水线

**五阶段流水线**：IF（取指）→ ID（译码）→ EX（执行）→ MEM（访存）→ WB（写回）

**流水线冲突**：
| 冲突类型 | 原因 | 解决方案 |
|----------|------|---------|
| 结构冲突 | 多个指令争用同一资源 | 增加资源、插入气泡 |
| 数据冲突 | 下一条依赖上一条结果 | 转发（旁路）、插入气泡 |
| 控制冲突 | 分支指令 | 分支预测、延迟分支 |

**数据冲突分类**：RAW（写后读）、WAR（读后写）、WAW（写后写）

**多发技术**：超标量、超流水线、超长指令字（VLIW）

#### 控制单元

| 特性 | 组合逻辑（硬布线） | 微程序 |
|------|:--------:|:--------:|
| 速度 | 快 | 慢 |
| 灵活性 | 差 | 好 |
| 设计成本 | 高 | 低 |
| 适用场景 | RISC | CISC |
| 修改难度 | 难（改硬件） | 易（改控制存储器） |

**微指令格式**：
- **水平型**：并行操作能力强，微指令字长长
- **垂直型**：并行能力弱，微指令字长短

**致命陷阱**：
| 陷阱 | 说明 |
|------|------|
| 数据冲突的 RAW 判断 | 后面指令读的寄存器，前面指令还没写完 |
| 分支预测失败 penalty | 预测失败时，已进流水线的指令全部作废 |
| 微程序 vs 硬布线 | 微程序=软件思路，硬布线=硬件逻辑门 |

**跨科缝合**：⛓️ → OS-进程上下文切换（流水线冲刷 ↔ 进程切换开销）

### 六、总线（🧱 砖墙）

**矛盾**：多个设备（CPU、内存、I/O）都要通信，但物理上只有一条共享通道。

**核心概念**：
- **总线分类**：片内总线、系统总线、通信总线
- **总线结构**：单总线、双总线（面向 CPU/面向存储器）、三总线、四总线
- **总线仲裁**：链式查询、计数器定时查询、独立请求
- **总线通信**：同步、异步（不互锁/半互锁/全互锁）、半同步、分离式

**三种仲裁对比**：
| 方式 | 速度 | 公平性 | 复杂度 |
|------|:----:|:------:|:------:|
| 链式查询 | 慢 | 差 | 低 |
| 计数器定时查询 | 中 | 中 | 中 |
| 独立请求 | 快 | 好 | 高 |

---

### 七、输入输出系统（🧱 砖墙）

**矛盾**：CPU 速度 GHz 级，外设速度毫秒级——CPU 怎么等得起？

**核心概念**：
- **I/O 控制方式**：程序查询、程序中断、DMA、通道、I/O 处理机
- **中断系统**：中断响应、中断优先级、中断屏蔽字
- **DMA 方式**：周期挪用、停止 CPU 访问、交替访问
- **I/O 接口**：数据线、设备选择线、命令线、状态线

**三种控制方式对比**：
| 方式 | CPU 参与 | 速度 | 适用设备 |
|------|:--------:|:----:|---------|
| 程序查询 | CPU 不断查询 | 慢 | 低速设备 |
| 程序中断 | CPU 响应中断 | 中 | 中速设备 |
| DMA | 不参与（硬件传送） | 快 | 高速设备（磁盘等） |

**中断 vs DMA**：
- 中断：每个指令周期结束后检查，需保存/恢复现场，适合低速设备
- DMA：DMA 控制器接管总线，直接在主存和 I/O 设备间传输，适合高速设备

### 致命陷阱清单

1. **Cache 地址拆分**：组号在中间，标记在高位，块内地址在低位
2. **补码运算**：符号位参与运算，溢出看最高位⊕次高位进位
3. **浮点数精度**：只有 2 的幂次组合能精确表示
4. **LRU 实现**：用顺序栈，不是时间戳
5. **数据冲突 RAW**：后面指令读的寄存器，前面指令还没写完
6. **分支预测失败**：已进流水线的指令全部作废
7. **微程序 vs 硬布线**：微程序是软件思路，硬布线是硬件逻辑门
8. **总线 vs 系统总线**：总线是广义概念，系统总线特指计算机内部连接 CPU/内存/I/O 的
