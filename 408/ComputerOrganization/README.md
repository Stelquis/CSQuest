## 一、计组为何存在？

### 核心矛盾

> **人用逻辑思考，机器用电流动。**
> 计组就是"翻译"这层矛盾的工程实现。

### 分层的本质

```
人的视角（程序员）
    ↓
高级语言（C / Python）
    ↓
指令系统（ISA）—— 计组学的"语言"
    ↓
微架构（数据通路 + 控制单元）
    ↓
数字电路（与或非门）
    ↓
晶体管（开关）
    ↓
物理世界（电子流动）
```

**计组回答的核心问题链：**

| 问题                           | 对应章节       | 矛盾本质                                                   |
| ------------------------------ | -------------- | ---------------------------------------------------------- |
| 数字怎么在机器里表示？         | 数据表示与运算 | 人的十进制 vs 机器的二进制，有限精度 vs 无限实数           |
| 指令怎么告诉机器该做什么？     | 指令系统       | 人的意图（操作）→ 机器的语言（二进制编码）                |
| CPU 怎么知道下一条指令是什么？ | CPU 与流水线   | 顺序执行 vs 并行加速，理想的"一直往前" vs 现实的"分支冲突" |
| 数据怎么从内存到 CPU？         | 存储器层次结构 | CPU 太快 vs 内存太慢，容量大 vs 速度快（不可能三角）       |
| 多个设备怎么通信？             | 总线           | 多个设备共享一条通道 vs 冲突仲裁                           |
| 机器怎么跟人交互？             | 输入输出系统   | 高速 CPU vs 低速外设，同步 vs 异步                         |

## 三、核心矛盾图谱（每章一个"为什么"）

### 第 1 章：计算机系统概述（🎨 涂料）

**矛盾：** 程序是逻辑，机器是电路——怎么把"如果-那么"变成"通-断"？

**核心概念：** 冯·诺依曼结构、存储程序、层次结构

**学习目标：** 建立"计算机是翻译机器"的全局视角，不深究细节。

**跨科缝合：** ⛓️ → OS-系统调用（用户态→内核态）

---

### 第 2 章：数据的表示与运算（🏗️ 梁柱）

**矛盾：** 人的数学是无限精度的，机器寄存器是有限位宽的。

**核心冲突链：**

```
有限位宽 → 溢出 → 如何判断溢出？
          → 浮点数精度损失 → 0.1 + 0.2 ≠ 0.3
          → 负数怎么表示？→ 原码/反码/补码
```

**核心概念：** 原码/反码/补码、浮点数 IEEE 754、ALU 运算、溢出判断

**致命陷阱：**

| 陷阱                 | 典型错误                     | 正确理解                         |
| -------------------- | ---------------------------- | -------------------------------- |
| 补码的符号位参与运算 | 以为符号位单独处理           | 符号位参与二进制加法，溢出看进位 |
| 浮点数精度           | 以为浮点数能精确表示所有小数 | 只有 2 的幂次组合能精确表示      |
| 溢出判断             | 只看最高位进位               | 需要看最高位进位 ⊕ 次高位进位   |

**跨科缝合：** ⛓️ → 数据结构-位图（位运算在数据压缩中的应用）

---

### 第 3 章：存储器层次结构（🏗️ 梁柱）

**矛盾：** CPU 太快，内存太慢——速度差了几个数量级。

**核心矛盾链：**

```
CPU 速度 (GHz) >> 内存速度 (MHz)
    → CPU 大部分时间在"等"数据
    → 解决方案：Cache（速度缓冲）+ 虚存（容量幻觉）
    → Cache 的核心问题：映射策略、替换算法、写入策略
    → 虚存的核心问题：页表、TLB、缺页处理
```

**核心概念：** SRAM/DRAM/ROM、Cache 映射（全相联/直接/组相联）、替换算法（LRU/FIFO/Random）、写入策略（写直达/写回）、虚拟存储器、页表、TLB

**必背公式：**

| 公式                                                      | 说明           |
| --------------------------------------------------------- | -------------- |
| 平均访问时间 = Hit × HitTime + Miss × MissPenalty       | Cache 性能核心 |
| Cache 容量 = 行数 × 每行大小（块大小 + 标记位 + 有效位） | 地址拆分计算   |
| 页表大小 = 页表项数 × 页表项大小                         | 虚存计算       |

**致命陷阱：**

| 陷阱                 | 说明                                                   |
| -------------------- | ------------------------------------------------------ |
| Cache 组号 vs 标记位 | 地址拆分时，组号位在中间，标记位在高位，块内地址在低位 |
| LRU 用堆栈实现       | 不是用时间戳，是用"最近使用"的顺序栈                   |
| TLB 与 Cache 的时序  | TLB 查完才查 Cache，不是同时查                         |

**跨科缝合：** ⛓️ → OS-内存管理-页表（Cache 组相联映射 ↔ 页表多级映射）；⛓️ → OS-页面置换算法（LRU 在 Cache 和 OS 中的实现差异）

---

### 第 4 章：指令系统（🏗️ 梁柱）

**矛盾：** 人的操作意图是"加两个数"，机器需要知道"用什么数、加到哪里、结果存哪"。

**核心矛盾链：**

```
人的操作意图（add a, b）
    → 指令格式：操作码 + 地址码
    → 寻址方式：立即数？寄存器？内存？
    → 指令长度：定长 vs 变长
    → CISC vs RISC：复杂指令 vs 精简指令
```

**核心概念：** 指令格式、寻址方式（立即/直接/间接/寄存器/基址/变址）、CISC vs RISC、指令周期

**必背：**

| 寻址方式   | 有效地址       | 速度 | 典型用途      |
| ---------- | -------------- | ---- | ------------- |
| 立即寻址   | 指令本身       | 最快 | 常数          |
| 寄存器寻址 | 寄存器         | 快   | 中间变量      |
| 直接寻址   | 主存地址       | 慢   | 全局变量      |
| 间接寻址   | 主存中存的地址 | 最慢 | 指针          |
| 基址寻址   | 基址 + 偏移    | 灵活 | 数组/结构体   |
| 变址寻址   | 变址 + 偏移    | 灵活 | 循环/数组遍历 |

**跨科缝合：** ⛓️ → OS-中断机制（异常指令的处理流程）

---

### 第 5 章：中央处理器（🏗️ 梁柱——大题常客）

**矛盾：** 顺序执行太慢——每条指令都等上一条完成，CPU 利用率极低。

**核心矛盾链：**

```
顺序执行 → 浪费大量时间在"等"
    → 流水线：把一条指令拆成多个阶段，重叠执行
    → 引入三个冲突：
        ① 结构冲突（硬件资源不够）
        ② 数据冲突（下一条依赖上一条的结果）
        ③ 控制冲突（分支指令——下一条在哪？）
    → 解决冲突：插入气泡、转发通路、分支预测
```

**核心概念：** 数据通路、硬布线/微程序控制器、指令流水线、流水线冲突（结构/数据/控制）、转发、分支预测

**流水线五阶段：**

```
IF（取指令）→ ID（译码）→ EX（执行）→ MEM（访存）→ WB（写回）
```

**致命陷阱：**

| 陷阱                 | 说明                                                                           |
| -------------------- | ------------------------------------------------------------------------------ |
| 数据冲突的 RAW 判断  | 不是"后面指令用到了前面指令的结果"，而是"后面指令读的寄存器，前面指令还没写完" |
| 分支预测失败 penalty | 预测失败时，已经进流水线的指令全部作废，不是只丢弃一条                         |
| 微程序 vs 硬布线     | 微程序=软件思路（ROM 存微指令），硬布线=硬件思路（逻辑门搭状态机）             |

**跨科缝合：** ⛓️ → OS-进程上下文切换（流水线冲刷 ↔ 进程切换的开销）

---

### 第 6 章：总线（🧱 砖墙）

**矛盾：** 多个设备（CPU、内存、I/O）都要通信，但物理上只有一条共享通道。

**核心概念：** 总线结构（单总线/多总线）、总线仲裁（链式/定时/独立请求）、总线标准

### 第 7 章：输入输出系统（🧱 砖墙）

**矛盾：** CPU 速度 GHz 级，外设速度毫秒级——CPU 怎么等得起？

**核心概念：** I/O 方式（程序查询/中断/DMA）、中断处理流程、DMA 传输

## 四、学习路线（照 learning-loop-system 走）

### 推荐顺序

```
第 1 章（涂料）→ 快速扫过，建立全局观
    ↓
第 2 章（梁柱）→ 数据表示，打好计算基础
    ↓
第 4 章（梁柱）→ 指令系统，理解机器语言
    ↓
第 3 章（梁柱）→ 存储层次，核心难点
    ↓
第 5 章（梁柱）→ CPU + 流水线，大题主力
    ↓
第 6 章（砖墙）→ 总线
    ↓
第 7 章（砖墙）→ 输入输出
```

### 每章执行 5 步（取自 learning-loop-system.md）

1. **定位矛盾** → 翻开本章前，先问"如果没有这个技术，计算机会出什么 bug？"
2. **定级** → 看王道"考情分析"，确定梁柱/砖墙/涂料
3. **预习** → 15 分钟扫标题、图表、加粗定义
4. **精读（三层加工）** → 只对梁柱级做全套
   - L1 结构化：画关系图（把概念按逻辑关系画出来）
   - L2 关联化：找隔壁邻居（标注 ⛓️ 跨科缝合点）
   - L3 模型化：抽象成一句话思维模型
5. **费曼检验** → 用大白话讲给小白听

### 跨科缝合点（提前标注，学到时注意）

| 计组章节              | 缝合科目 | 缝合点             | 建议学习时机                |
| --------------------- | -------- | ------------------ | --------------------------- |
| Cache（第 3 章）      | OS       | 页表、虚拟内存     | 学完两者后，找一道联考题    |
| 指令流水线（第 5 章） | OS       | 进程上下文切换     | 学完流水线 + 学完进程调度后 |
| 中断（第 7 章）       | OS       | 中断处理、系统调用 | 学完两者后对比              |
| TLB（第 3 章）        | OS       | 页表查询           | 同时学，一次搞懂            |

## 五、实操建议（来自 learning-loop-system.md）

### 状态匹配

| 身体状态 | 推荐模式    | 做什么                                          |
| -------- | ----------- | ----------------------------------------------- |
| 充满电   | 🔥 满血模式 | 学第 5 章流水线（新知识）、画数据通路图、做大题 |
| 有点沉   | 🚙 巡航模式 | 刷第 2 章计算题、整理第 3 章 Cache 公式         |
| 极度抗拒 | 🛵 苟活模式 | 抄写 Cache 地址拆分步骤、重抄一遍流水线五阶段   |

### 避坑

- ❌ 不要从第 5 章 CPU 开始（前置要求高，容易卡死）
- ❌ 不要第 3 章和第 5 章同时学（都是梁柱级，脑容量不够）
- ✅ 第 2 章（数据表示）是最安全的起点
- ✅ 第 2 章的计算题每天练 2 道，保持手感

### 十五分钟热试

每天翻开书，先抄 3 行 Cache 公式或画一遍流水线五阶段，15 分钟后身体会告诉你今天该用什么模式。
