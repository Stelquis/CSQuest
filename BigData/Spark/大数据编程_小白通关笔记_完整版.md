## 二、`_`（下划线）的所有用法

`_` 是 Scala/Spark 的万能符号，不同位置意思不同。

### 用法1：参数占位符（最常用，Spark代码里到处是）

**一个 `_` = 一个参数**：

```scala
map(x => x * 2)              ← 完整写法
map(_ * 2)                   ← 简写

filter(x => x.length > 0)    ← 完整写法
filter(_.length > 0)         ← 简写

flatMap(line => line.split(" "))  ← 完整
flatMap(_.split(" "))             ← 简写
```

**两个 `_` = 两个参数**：

```scala
reduceByKey((x, y) => x + y)   ← 完整
reduceByKey(_ + _)              ← 简写

// 第一个 _ = x，第二个 _ = y
```

**怎么记**：`_` 就是"那个东西"，1个就一个，2个就两个，从左到右对应。

### 用法2：match-case 通配符 = default

```scala
x match {
    case 1 => "one"
    case _ => "many"    // 除了1以外的所有情况
}
```

### 用法3：元组取值

```scala
val pair = ("a", 1)
pair._1    // "a"（第1个）
pair._2    // 1  （第2个）
```

### 用法4：导包重命名/隐藏

```scala
import java.util.{ HashMap => JavaHashMap }    // 改名
import java.util.{ HashMap => _, _ }           // 隐藏HashMap
```

### 用法5：for循环忽略值

```scala
for (_ <- 1 to 5) println("hi")   // 不需要用 i
```

## 三、元组（Tuple）——从零讲起

**元组就是"把几个东西捆在一起"**。

```scala
val t = ("珍珠奶茶", 15, 5.0)
//           ↑第1个  ↑第2个 ↑第3个

t._1    // "珍珠奶茶"
t._2    // 15
t._3    // 5.0
```

**关键规则**：元组**从1开始**（不是0）。

**嵌套元组**（第五章代码）：

```scala
val x = (95, ("95031", "1001"))
//        第1   第2（它本身又是元组）

x._1      = 95
x._2      = ("95031", "1001")
x._2._1   = "95031"
x._2._2   = "1001"
```

**元组 vs 数组**：

| | 元组 | 数组 |
|:--|:---:|:---:|
| 索引从？ | **1** | **0** |
| 类型必须一样？ | ❌ 可以不同 | ✅ 必须一样 |
| 长度 | 固定 | 可变化 |

**解构赋值**：
```scala
val (name, price, score) = ("奶茶", 15, 5.0)
// name="奶茶", price=15, score=5.0
```

## 四、Unit vs Int——到底什么区别

```scala
def increase(): Unit = { age += 1 }    // 只做事，不给东西
def current(): Int = { age }           // 做事，还给个值
```

**判断方法**：看最后一行
- 最后一行是**操作**（`age += 1`）→ `Unit`（没返回值）
- 最后一行是**值**（`age`）→ `Int`（有返回值）

**Unit 的本质**：就是 Java 的 `void`。Scala 是纯面向对象语言，连"没有值"也得是个类型，所以搞了个 `Unit`，它只有唯一的值 `()`。

```scala
val x = increase()    // x: Unit = ()   ← 就是个空括号，没有意义
val y = current()     // y: Int = 19    ← 一个真正的数字
```

## 第一章 Spark概述

> ⭐⭐⭐⭐⭐ 百分之百会考

### Spark是什么？

**一句话**：一个比 Hadoop MapReduce **快100倍**的"超级计算器"。

### 发源地

**2009年**，加州大学伯克利分校 **AMP实验室**（Algorithms, Machines and People Lab），由 Lester 和 Matei 创建。

- 名字"Spark" = 电光火石 = 快
- 内存速度可达 Hadoop 的 **100多倍**

### 六大特点（用六个字记：快、稳、多、简、通、全）

| 特点 | 一句话 |
|:----|-------|
| **快** | 用 DAG 执行计划，中间结果在内存，不是所有场景都需要排序 |
| **稳** | 引入 RDD，某台机器挂了可以重算恢复 |
| **多** | 不止 Map/Reduce 两种操作，有 map/filter/flatMap/reduceByKey... |
| **简** | 支持 Scala/Python/Java/R 四种语言，代码量比 Hadoop 少 50%~80% |
| **通** | 可以用 YARN/Mesos/Standalone 模式，能读 HDFS/Cassandra/HBase |
| **全** | 批处理 + 流处理 + 交互查询 + 图计算 + 机器学习一套搞定 |

### 生态系统（课本第4页图）

```
        Spark SQL     Streaming      GraphX
        (表格数据)     (实时流)       (图计算)     ← 上层应用
            MLlib（机器学习）
        ───────────────────────
              Spark Core
          (核心引擎：任务调度、内存管理)             ← 核心
        ───────────────────────
          YARN / Mesos / Standalone
              HDFS / HBase / Cassandra           ← 基础设施
```

### 运行架构

**核心术语**：

| 术语 | 大白话 | 像什么 |
|:----|-------|:-----|
| Driver | 总指挥 | 包工头 |
| SparkContext | 对讲机 | 包工头的对讲机 |
| Executor | 工人 | 每个工位上干活的 |
| Task | 任务单 | 包工头发的小纸条 |
| Stage | 工作阶段 | 按Shuffle切开的阶段 |
| Shuffle | 数据跨机器传输 | 把材料从A工地运到B工地 |

**运行流程**：
```
Application → Job(一个Action触发) → Stage(Shuffle边界) → Task(一个分区)
```

**五大运行模式**：
| 模式 | 填什么 |
|:----|:-----:|
| 本地模式 | `local` |
| 本地多线程 | `local[N]` 或 `local[*]` |
| Standalone | `spark://host:port` |
| YARN | `yarn` |
| Mesos | `mesos://host:port` |

### WordCount——全书最经典案例

```scala
import org.apache.spark.{SparkConf, SparkContext}

object WordCount {
    def main(args: Array[String]): Unit = {
        val conf = new SparkConf().setMaster("local").setAppName("wordcount")
        val sc = new SparkContext(conf)

        val lines = sc.textFile("./src/word")        // 读文件
        val words = lines.flatMap(_.split(" "))      // 拆单词
        val pairs = words.map((_, 1))                // 标1
        val count = pairs.reduceByKey(_ + _)         // 聚合
        count.collect().foreach(println)             // 输出
    }
}
```

**数据流转**：
```
文件内容："a b","d e","c c","a c","e e"
  → flatMap: ["a","b","d","e","c","c","a","c","e","e"]
  → map: [("a",1),("b",1),...]
  → reduceByKey: [("a",2),("b",1),("c",3),("d",1),("e",3)]
  → collect输出
```

## 第三章 Scala语言基础

> ⭐⭐⭐ 较重要，不会单独考概念，但会给代码让你算结果

### 三种变量

| 关键字 | 能否改 | 什么时候赋值 | 像Java的 |
|:------|:-----:|:-----------:|:--------:|
| `val` | ❌ 不能 | 定义时 | `final` |
| `var` | ✅ 能 | 随时 | 普通变量 |
| `lazy val` | ❌ 不能 | **用的时候**才赋值 | Java没有 |

**易错点**：
- `var` 可改值但**不能改类型**
- `lazy` 只能修饰 `val`
- Scala 自动推断类型

### if-else 是表达式（和Java最大的不同！）

```scala
// Java：if-else 没有返回值
int result;
if (x > 0) result = 1; else result = -1;

// Scala：if-else 有返回值，可以直接赋值
val result = if (x > 0) 1 else -1   // ✅
```

### for 循环

```scala
for (i <- 1 to 5) println(i)         // 1,2,3,4,5（包含）
for (i <- 1 until 5) println(i)      // 1,2,3,4（不包含）
for (i <- 1 to 5 if i > 3) println(i)  // 带守卫：只输出4,5

// for-yield：生成新集合 ⭐
val doubled = for (i <- 1 to 5) yield i * 2
// 结果：Vector(2, 4, 6, 8, 10)
```

### match-case（加强版switch）

```scala
// 基本匹配
def test(x: Int): String = x match {
    case 1 => "one"
    case 2 => "two"
    case _ => "many"          // _ = 通配符
}

// 类型匹配 ⚠️ switch做不到
def test(x: Any): Any = x match {
    case y: Int => "Int类型"
    case s: String => "String类型"
    case _ => "其他"
}

// 带守卫
x match {
    case _ if x % 2 == 0 => "偶数"
    case _ => "奇数"
}
```

### 集合

| 集合 | 要点 |
|:----|------|
| **Array** | 访问用 `()` 不是 `[]`，索引从0开始 |
| **List** | 不可变，`::` 连接，`head`/`tail` 取元素 |
| **Set** | 自动去重，`min`/`max` 取最值 |
| **Map** | `("a" -> 1)` 创建，`keys`/`values` 遍历 |
| **Tuple** | ⚠️ **从1开始**：`t._1` 是第一个 |
| **Option** | `Some(值)` 或 `None`，`getOrElse` 安全取值 |

### Iterator（迭代器）

```scala
val iter = Iterator("Hadoop", "Spark", "Scala")
while (iter.hasNext) println(iter.next())
// ⚠️ 一次性！遍历完就没了
```

### 递归 vs 尾递归

```scala
// 普通递归：一直记着要回来，容易栈溢出
def factorial(n: BigInt): BigInt = {
    if (n <= 1) 1
    else n * factorial(n - 1)
}

// 尾递归：不记着回来，编译器优化成循环 ⚠️
@annotation.tailrec
def factorial(n: Int, acc: Int): Int = {
    if (n <= 1) acc
    else factorial(n - 1, n * acc)
}
```

### 高阶函数（函数也能当参数）

```scala
def apply(f: Int => String, v: Int) = f(v)
def layout[A](x: A) = "[" + x.toString() + "]"
println(apply(layout, 10))   // 输出：[10]
```

## 第四章 Scala面向对象编程

> ⭐⭐⭐ 较重要

### 类的基本定义

```scala
class Student {
    private var age = 18    // 私有
    val name = "Scala"       // 公有（默认）
    def current(): Int = { age }

```

### 主构造函数（参数写在类名后面）

```scala
class Student(val name: String, val classNum: Int) {
    // 加 val/var 自动变成成员变量
    private var age = 18

```

### 辅助构造函数（第一行必须调 `this()`）⚠️

```scala
class Student {
    var name = ""
    def this(name: String) {
        this()              // ⚠️ 第一行必须是 this()
        this.name = name
    }
```

### object 单例对象

- `object` = 全局唯一，**不能 `new`**
- 像 Java 的 `static` 方法

### 伴生对象（class + object 同名同文件）

- 可以互相访问**私有成员**

### 抽象类

```scala
abstract class Phone {
    val phoneBrand: String      // 抽象变量（必须声明类型）
```

- 重写非抽象方法**必须加 `override`**
- 重写抽象方法可以不加
- 构造顺序：**先父类，再子类**
- **动态绑定**：调哪个方法看运行时实际类型，不是声明类型

| | Trait | Abstract Class |
|:--|:----:|:-------------:|
| 多继承 | ✅ 可以混入多个 | ❌ 只能一个 |
| 构造参数 | ❌ 不能 | ✅ 可以 |


## 第五章 RDD编程——全书最重要

> ⭐⭐⭐⭐⭐ **"非常重要！后面所有编程都基于RDD"**

### RDD = 弹性分布式数据集 = 切成多块散落在多台机器上的大数组

三个关键词：
- **Resilient（弹性）**：机器挂了可以重算恢复
- **Distributed（分布式）**：存在多台机器上
- **Dataset（数据集）**：一堆数据

### 两种操作（最核心的概念）

**Transformation（转换操作）**：
- 特点：**不立即执行**，只记录，返回新RDD
- 像什么：你列了个购物清单，还没去买
- 有哪些：`map`、`flatMap`、`filter`、`reduceByKey`、`groupByKey`、`sortByKey`、`join`

**Action（行动操作）**：
- 特点：**触发计算**，返回结果或存文件
- 像什么：你喊"开工了！"
- 有哪些：`collect`、`count`、`first`、`take`、`saveAsTextFile`

**⚠️ 最易错的坑**：
| 操作 | 名字有reduce | 是T还是A？ |
|:----|:----------:|:---------:|
| `reduceByKey` | ✅ | **T**（返回RDD）→ 所有ByKey结尾的都是T |
| `reduce` | ✅ | **A**（返回一个值） |

### 常用操作

**Value型**：
- `map` → 一对一映射
- `flatMap` → 一对多+拍平（WordCount拆词用这个）
- `filter` → 保留符合条件的
- `distinct` → 去重
- `sortBy` → 排序

**Key-Value型**：
- `reduceByKey` → 按Key聚合，**有Map端预聚合**（快）
- `groupByKey` → 按Key分组，**无预聚合**（慢）
- `mapValues` → 只改Value不改Key
- `join` → 两个RDD按Key关联
- `sortByKey` → 按Key排序

### reduceByKey vs groupByKey（重点！）

```scala
// reduceByKey：先本地加一遍再传网络（快）
rdd.reduceByKey(_ + _)

// groupByKey：原样传网络（慢，可能OOM）
rdd.groupByKey().mapValues(_.sum)
```

**一句话**：`reduceByKey` 有**本地预聚合**，`groupByKey` 没有。

### 窄依赖 vs 宽依赖（Stage划分依据）

| | 窄依赖 | 宽依赖 |
|:--|:-----:|:-----:|
| 意思 | 父分区最多被子分区用一次 | 父分区被子分区用多次 |
| 需要Shuffle？ | ❌ 不需要 | ✅ 需要 |
| 有哪些？ | map/filter/flatMap/union | reduceByKey/groupByKey/join |
| 像什么？ | 各干各的不用找人 | 得把不同机器的数据凑一起 |

### 两个综合实例

**实例1：成绩排名**
```
数据格式：序号,学号,成绩,班级
操作链：filter → map(提取字段) → sortByKey(false) → take(10)
```

**实例2：基站数据分析**
```
核心技巧：连接(1)取负时间，断开(0)取正时间 → reduceByKey求和 = 停留时长
操作链：map → reduceByKey → map(转换结构) → join(基站信息)
```

## 第六章 Spark SQL与DataFrame

> ⭐⭐⭐⭐⭐ **大题最可能来源**

### DataFrame = 有列名+类型的"分布式Excel表"

```
RDD： ["MI", "20", "china"]           ← 不知道每列是什么
DF：  name=MI, age=20, country=china  ← 知道列名和类型
```

### SparkSession（入口变了）

```scala
val spark = SparkSession.builder()
    .master("local").appName("SQL").getOrCreate()
import spark.implicits._   // ⚠️ 必须导入！
```

### DataFrame创建的5种方式

1. **JSON**：`spark.read.json("x.json")` — 自动推断Schema
2. **Parquet**：`spark.read.parquet("x.parquet")` — 默认格式
3. **Case Class**：`rdd.toDF()` — ⚠️ 代码补全最爱
4. **StructType**：`spark.createDataFrame(rowRDD, schema)` — 灵活
5. **JDBC**：`spark.read.format("jdbc").option(...)` — 连MySQL

**Case Class方式详解**：
```scala
case class Student(name: String, age: Int, height: Int)
val df = sc.textFile("x.txt").map(_.split(","))
    .map(f => Student(f(0), f(1).toInt, f(2).toInt)).toDF()
df.createOrReplaceTempView("students")
spark.sql("SELECT * FROM students WHERE age > 20").show()
```

### DataFrame操作

```scala
df.show()                    // 显示
df.printSchema()             // 打印结构
df.select("name","age")      // 选列
df.filter($"age" > 20)       // 过滤（⚠️ 用 $ 引用列）
df.groupBy("country").count()
df.orderBy($"age".desc)
df1.join(df2, df1("id") === df2("id"), "inner")  // ⚠️ 注意 ===
df.createOrReplaceTempView("t")  // 注册后可以写SQL
```

### DataFrame vs RDD（简答题高频）

> RDD 没结构 → 不能优化
> DataFrame 有Schema → Catalyst能自动优化（列裁剪、谓词下推）

### Catalyst优化器流程


## 第七章 Spark Streaming流式计算

> ⭐⭐⭐⭐⭐ **3种特殊数据之二：流式数据**

### 核心思想

数据一点一点来（串行）→ 攒一批 → 一起处理（并行）

**DStream = RDD的序列**，每隔一个时间间隔产生一个RDD。

### StreamingContext

```scala
val conf = new SparkConf().setMaster("local[2]").setAppName("Streaming")
//                                           ↑ 至少2线程，1收1处理
val ssc = new StreamingContext(conf, Seconds(10))
//                                  ↑ 批处理间隔10秒
ssc.start()
ssc.awaitTermination()
```

### 4种输入源

| 输入源 | 写法 |
|:------|------|
| Socket | `ssc.socketTextStream(host, port)` |
| 文件 | `ssc.textFileStream(dir)` |
| RDD队列 | `ssc.queueStream(rddQueue)` |
| Kafka | `KafkaUtils.createStream(ssc, zk, group, topics)` |

### 窗口操作（难点！）

```
窗口长度 = 看多久的数据（如30秒）
滑动间隔 = 每隔多久算一次（如10秒）

时间轴：0    10    20    30    40
窗口1： [============]           ← 第0~30秒
窗口2：        [============]    ← 第10~40秒
```

```scala
// 基本窗口
words.window(Seconds(30), Seconds(10))

// 高效版reduceByKeyAndWindow ⚠️ 需要checkpoint
words.map(x => (x, 1))
    .reduceByKeyAndWindow(_ + _, _ - _, Seconds(30), Seconds(10))
// 只算新增和移除的，不需要重算整个窗口
```

### updateStateByKey（有状态操作）

```scala
// ⚠️ 必须设checkpoint！
ssc.checkpoint("/ckpt")

words.map((_, 1)).updateStateByKey(
    (currValues: Seq[Int], preValue: Option[Int]) => {
        Some(currValues.sum + preValue.getOrElse(0))
        //   当前批次的和 + 历史累计值
    }
)
```

**3个要点**：
1. 必须设checkpoint
2. 第一个参数是当前批次值的Seq，第二个参数是历史Option
3. 返回值必须包在 `Some()` 里

### 窗口 vs updateStateByKey

| | 窗口 | updateStateByKey |
|:--|:---:|:--------------:|
| 看多久？ | 最近一段时间（如30秒） | 从开始到现在 |
| 需要checkpoint？ | 高效版需要 | ✅ 必须 |
| 像什么？ | 热搜榜 | 历史总榜 |

## 第八章 Spark GraphX图计算

> ⭐⭐⭐⭐⭐ **三种特殊数据之三：图数据**

### 图 = 顶点（Vertex）+ 边（Edge）

```scala
// 顶点：(ID, 属性)，ID必须是Long
val vertices = sc.parallelize(Array(
    (1L, "Tom"), (2L, "Marry")
))

// 边：Edge(源ID, 目标ID, 属性)
val edges = sc.parallelize(Array(
    Edge(1L, 2L, "同事")
))

// 建图
val graph = Graph(vertices, edges, "defaultUser")
```

### 图的基本属性

```scala
graph.vertices       // 所有顶点
graph.edges          // 所有边
graph.numVertices    // 顶点数
graph.numEdges       // 边数
graph.degrees        // 度（入度+出度）
graph.inDegrees      // 入度（被别人指向的次数）
```

### aggregateMessages（顶点之间传消息）

```scala
graph.aggregateMessages[Int](
    triplet => {
        triplet.sendToDst(1)
        triplet.sendToSrc(1)
    },
    (a, b) => a + b    // 合并消息
)
```

### Pregel API（迭代式传播）

```scala
graph.pregel(初始消息, 最大迭代次数)(
    vprog,    // 收到消息怎么更新自己
    sendMsg,  // 更新完给邻居发什么
    mergeMsg  // 多条消息怎么合并
)
```

### 内置算法

```scala
graph.pageRank(0.0001).vertices          // PageRank
graph.triangleCount().vertices           // 三角形计数
```

### 点分割 vs 边分割

GraphX 用**点分割**——每条边存一次，顶点可以重复。
原因：**磁盘便宜，网络带宽贵**，多存一分别多传一次。

## 第九章 Spark机器学习原理

> ⭐⭐⭐⭐⭐ **百分之百会考**

### spark.ml vs spark.mllib

考试用**spark.ml**（基于DataFrame的），spark.mllib（基于RDD的）已过时。

### Pipeline（流水线）的核心概念

```
原始数据 → Transformer → Transformer → Estimator.fit() → PipelineModel
                                          ↓
新数据   → PipelineModel.transform() → 预测结果
```

**两种"零件"**：

| 角色 | 方法 | 像什么 |
|:----|:---:|:------|
| **Transformer** | `transform()` | 榨汁机：苹果进苹果汁出 |
| **Estimator** | `fit()` | 厨师：学完菜谱变成"会做菜的厨师" |

**参数设置两种方式**：
1. setter：`lr.setMaxIter(10).setRegParam(0.01)`
2. ParamMap：`ParamMap(lr.maxIter -> 20)`

### Pipeline完整示例

```scala
val tokenizer = new Tokenizer().setInputCol("text").setOutputCol("words")
val hashingTF = new HashingTF().setNumFeatures(1000).setOutputCol("features")
val lr = new LogisticRegression().setMaxIter(10)

val pipeline = new Pipeline().setStages(Array(tokenizer, hashingTF, lr))
val model = pipeline.fit(training)
model.transform(test).select("prediction").show()

// 保存和加载
model.write.overwrite().save("/path")
val sameModel = PipelineModel.load("/path")
```

### 特征工程

| 转换器 | 干什么 |
|:------|--------|
| TF-IDF | 计算词的重要程度（词频×逆文档频率） |
| Word2Vec | 把词变成向量，意思相近的向量也相近 |
| Binarizer | 连续值变二值（超阈值=1） |
| MinMaxScaler | 归一化到[0,1] |
| StringIndexer | 文字标签变数字 |
| VectorAssembler | 多列合成一列 |
| VectorSlicer | 从向量中提取一部分 |
| RFormula | 用R语言公式写特征 |
| ChiSqSelector | 挑出最有用的特征 |

### 模型选择（调参）

```scala
// 参数网格：3×2=6种组合
val paramGrid = new ParamGridBuilder()
    .addGrid(hashingTF.numFeatures, Array(10, 100, 1000))
    .addGrid(lr.regParam, Array(0.1, 0.01)).build()

// 交叉验证：分成K份，轮流做验证集
val cv = new CrossValidator()
    .setEstimator(pipeline)
    .setEvaluator(new BinaryClassificationEvaluator)
    .setEstimatorParamMaps(paramGrid)
    .setNumFolds(2)

// 训练-验证拆分：一次性拆成80%训练20%验证
val tvs = new TrainValidationSplit()
    .setTrainRatio(0.8)
```

## 第十章 Spark机器学习模型

> ⭐⭐⭐⭐⭐ 需要真正用代码实现ML算法

### 通用模板（所有算法都一样）

```scala
val data = spark.read.format("libsvm").load("x.txt")
val Array(train, test) = data.randomSplit(Array(0.7, 0.3))
val model = new 算法名().setXxx(x).fit(train)
val predictions = model.transform(test)
val evaluator = new 评估器().setMetricName("指标")
val score = evaluator.evaluate(predictions)
```

### 4类问题

| 问题 | 输出 | 算法 | 评估指标 |
|:----|:---:|:----|:--------|
| **分类** | 是/否、类别 | 朴素贝叶斯、决策树分类 | accuracy |
| **回归** | 数字 | 线性回归、决策树回归 | RMSE、R² |
| **聚类** | 分到哪一组 | K-Means | silhouette（轮廓系数） |
| **关联规则** | 什么和什么一起出现 | FP-Growth | support/confidence |

### 各算法要点

**朴素贝叶斯**：
$$P(类别|特征) = \frac{P(特征|类别) \times P(类别)}{P(特征)}$$

"朴素"假设：所有特征相互独立。

**线性回归**：
- `ElasticNetParam`: 0=L2(Ridge), 1=L1(Lasso), 中间=混合
- 模型摘要：`coefficients`(系数)、`intercept`(截距)、`RMSE`、`r2`

**K-Means**：
1. 随机选K个中心点
2. 各点分配到最近的中心
3. 重新计算中心
4. 重复2-3到收敛

**FP-Growth**：
- 支持度：这个组合在多少购物篮里出现
- 置信度：买了A的人多少也买了B

### 评估指标

| 指标 | 人话 | 范围 |
|:----|------|:---:|
| AUC | 区分好人和坏人的能力 | [0,1]，越大越好 |
| Accuracy | 猜对的÷总数 | [0,1]，越大越好 |
| RMSE | 预测平均差多少 | [0,∞)，越小越好 |
| R² | 模型解释了多少数据变化 | [0,1]，越大越好 |
| Silhouette | 同类近、不同类远的程度 | [-1,1]，越大越好 |

### 核心易错点

1. **元组从1开始**（`t._1`），数组/列表从0开始
2. **`===` 不是 `==`**（DataFrame的join条件用 `===`）
3. **辅助构造函数第一行必须 `this()`**
4. **`reduceByKey` 是T不是A**（所有ByKey结尾的都是T）
5. **DataFrame要 `import spark.implicits._`**
6. **Streaming的 `local[2]` 至少2线程**
7. **窗口高效版和updateStateByKey都要checkpoint**
8. **顶点ID是Long类型**（后面加L）
9. **`override` 重写非抽象方法必须加**
10. **if-else在Scala里是表达式有返回值**
