## 六、核心数据结构

### 6.1 Block（区块）

```python
@dataclass
class Block:
    index: int                    # 区块索引（创世区块=0）
    transactions: List[Transaction]  # 交易列表
    timestamp: float              # Unix 时间戳
    previous_hash: str            # 前一区块哈希（链式连接）
    nonce: int = 0                # PoW 工作量证明随机数
    hash: str = ""                # 当前区块 SHA-256 哈希
    validator: str = ""           # PoS 验证者节点 ID（PoW 时为空）
```

哈希计算：`SHA-256(json(index, transactions, timestamp, previous_hash, nonce, validator))`

### 6.2 Transaction（交易）

```python
@dataclass
class Transaction:
    sender: str        # 发送方
    receiver: str      # 接收方
    amount: float      # 金额
    timestamp: float   # 时间戳
```

### 6.3 Node（PoS 节点）

```python
@dataclass
class Node:
    node_id: str           # 节点标识
    stake_amount: float    # 质押代币数量
    creation_time: float   # 注册时间（用于计算在线时长）
    is_active: bool = True # 活跃状态
```

## 七、共识算法详解

### 7.1 PoW — 工作量证明

**挖矿算法**：
```
输入：block(未完成), difficulty=4
目标：hash[:4] == "0000"

while block.hash[:difficulty] != "0" * difficulty:
    block.nonce += 1
    block.hash = block.calculate_hash()

返回：挖矿完成的区块（含有效 nonce 和哈希）
```

**验证函数** `verify_block()`：
1. 重算哈希 → 与 `block.hash` 比对
2. 检查 `hash[:difficulty] == "0" * difficulty`
3. 检查 `block.previous_hash == previous_block.hash`
4. 检查 `block.index == previous_block.index + 1`

**统计信息**：平均挖矿时间、最大/最小挖矿时间、平均 Nonce 值、难度

### 7.2 PoS — 权益证明

**权重计算**：
```
weight = stake_amount × (1 + min(online_hours, 100))

设计理由：
- stake_amount：质押越多，权重越大（经济激励）
- time_factor (1 + min(hours, 100))：在线越久越可信，上限 100h 防过度集中
```

**轮盘赌选择算法**：
```python
total_weight = sum(node.weight for node in active_nodes)
random_point = random.uniform(0, total_weight)  # [0, total)

current_sum = 0
for node in active_nodes:
    current_sum += node.weight
    if current_sum >= random_point:
        return node  # 被选中！
return active_nodes[-1]  # 兜底
```

**选中概率**：`P(node_i) = weight_i / Σ(weight_j)`

**出块奖励**：
```python
reward = (10.0 + stake_amount / 1000) × reward_coefficient
```

**验证函数** `verify_block()`：
1. 验证者不为空且为活跃节点
2. 验证者质押量 ≥ 最低门槛
3. 重算哈希比对

## 八、AI 参数优化模块

### 8.1 优化目标

| 参数 | 范围 | 优化方向 | 说明 |
|------|:----:|----------|------|
| `min_stake` | 50-500 | 安全性 ↔ 参与度 | 门槛高→安全但参与少 |
| `reward_coefficient` | 0.5-2.0 | 激励 ↔ 成本 | 高激励→多节点但通胀 |

### 8.2 模型设计

**主模型：决策树回归 (DecisionTreeRegressor)**
- 特征：`[node_count, avg_stake, tps]` — 网络当前状态
- 目标1：`optimal_min_stake` — 最优质押门槛
- 目标2：`optimal_reward_coefficient` — 最优奖励系数
- 参数：`max_depth=5, random_state=42` — 防过拟合
- 优点：可解释性强，无需特征缩放，能捕捉非线性关系

**备选方案：规则引擎**（sklearn 不可用时自动降级）
```python
# 节点数 → 最低质押
if node_count < 10:    min_stake = 100
elif node_count < 50:  min_stake = 200
else:                  min_stake = 300

# TPS → 奖励系数
if tps < 10:           reward = 1.5
elif tps < 50:         reward = 1.0
else:                  reward = 0.8
```

### 8.3 工作流程

```
1. collect_data()    → 采集 (node_count, avg_stake, tps) 样本
2. _generate_simulated_data() → 样本不足5条时生成10条模拟数据
3. train_model()     → 训练两个 DecisionTreeRegressor
4. predict_parameters() → 输入当前状态 → 输出推荐参数
5. 优化历史记录      → optimization_history 保存每次优化结果
```

## 九、链验证与安全设计

### 9.1 全链验证 `validate_chain()`

```
for i in [1, len(chain)):
    ① 验证哈希链接:   block[i].previous_hash == block[i-1].hash
    ② 验证索引连续:   block[i].index == block[i-1].index + 1
    ③ 验证区块哈希:   block[i].hash == block[i].calculate_hash()
    ④ 根据类型验证:
       - PoW 区块 → hash[:difficulty] 满足前导零要求
       - PoS 区块 → 验证者合法 + 质押量 ≥ 门槛
```

### 9.2 安全性评分

| 共识 | 评分公式 | 说明 |
|------|----------|------|
| PoW | `min(100, difficulty × 20)` | 难度4 → 80分，难度越高越安全 |
| PoS | `min(100, node_count × 15)` | 4节点 → 60分，节点越多越安全 |
