    │     ├── build_merkle_tree()        构建默克尔树（支持1~N文档）

    │     ├── encrypt_digest()           OAEP 填充加密
    │     └── decrypt_digest()           RSA 解密（含错误密钥测试）
    │
    ├── 模块三: 数字签名模块  ── RSA-PSS 签名与验证
    │     ├── create_notarization_record()  构建存证记录
    │     ├── sign_record()                 RSA-PSS 签名
    │     └── verify_signature()            5种场景验证
    │
    └── 模块四: 身份验证      ── 学号 SHA-256 哈希绑定
          └── compute_student_hash()    学号 → 唯一身份标识

### 5.2 数据流

```
文档创建 → SHA-256 哈希 → Merkle Tree 构建 → 根哈希比对（完整性验证）
    │
    ▼

    │
    ▼

    │
    ▼
学号 → SHA-256 哈希 → 身份绑定 → 总结报告输出
```

## 六、核心算法说明

### 6.1 SHA-256 哈希计算

```python
def compute_file_hash(content):
    return hashlib.sha256(content.encode("utf-8")).hexdigest()
```
- 输入：文档内容字符串
- 输出：64位十六进制哈希值
- 雪崩效应：任意1 bit 变化导致约50%哈希位翻转

### 6.2 默克尔树构建（支持 N 文档）

1. 叶子层：计算所有文档的 SHA-256 哈希
2. 奇数处理：节点数为奇数时复制最后一个节点
3. 向上归并：逐层两两拼接计算父哈希
4. 根哈希：最终单一哈希值为默克尔根，作为完整性基准

时间复杂度：O(n log n)，空间复杂度：O(n)

### 6.3 RSA-2048 加密方案

| 参数 | 取值 | 说明 |
|------|------|------|
| 密钥长度 | 2048 bit | 约 112 位安全强度 |
| 公钥指数 | 65537 (F4) | 标准推荐值，安全且高效 |
| 填充方案 | OAEP | 选择密文攻击防护 |
| 哈希算法 | SHA-256 | OAEP 内部摘要 |
| 掩码生成 | MGF1(SHA-256) | 标准掩码生成函数 |

### 6.4 RSA-PSS 数字签名方案

| 参数 | 取值 | 说明 |
|------|------|------|
| 摘要算法 | SHA-256 | 消息摘要 |
| 掩码生成 | MGF1(SHA-256) | 标准掩码函数 |
| 盐长度 | PSS.MAX_LENGTH | 最大安全性 |
| 输出编码 | Base64 | 可传输文本格式 |

## 七、功能说明

### 7.1 模块一：文档创建与完整性验证

| 函数 | 功能 | 亮点 |
|------|------|------|
| `create_documents()` | 创建4个模拟AI专利文档 | 模拟 AI 生成内容存证场景 |
| `compute_file_hash(content)` | SHA-256 哈希计算 | 字符串输入 |
| `compute_file_hash_from_path(path)` | 大文件分块哈希 | **扩展**：支持图片/PDF（8KB分块） |
| `build_merkle_tree(hashes, verbose)` | 构建默克尔树 | **扩展**：支持1~N任意数量文档 |
| `verify_integrity(docs, root)` | 完整性验证 | 重算根哈希比对 |
| `detect_tampered_document(...)` | 智能篡改定位 | **扩展**：指出具体被篡改文档编号 |

### 7.2 模块二：安全传输模块

| 函数 | 功能 |
|------|------|
| `generate_rsa_keys()` | 生成 RSA-2048 密钥对 |
| `serialize_public_key(key)` | 公钥序列化为 PEM 格式 |
| `encrypt_digest(pub_key, digest)` | OAEP 加密文档摘要 → Base64 密文 |
| `decrypt_digest(priv_key, ciphertext)` | 解密还原摘要 → 错误密钥抛异常 |

验收标准：
- ✅ 加密解密后哈希与原值一致
- ✅ 错误私钥解密抛出异常

### 7.3 模块三：数字签名与存证

| 函数 | 功能 |
|------|------|
| `create_notarization_record(...)` | 创建含完整元数据的存证记录 |
| `sign_record(priv_key, record)` | JSON 序列化后 RSA-PSS 签名 → Base64 |
| `verify_signature(pub_key, record, sig)` | 反序列化验证，捕获 InvalidSignature |

### 7.4 模块四：系统集成

| 函数 | 功能 |
|------|------|
| `compute_student_hash(id)` | SHA-256(学号) → 唯一身份绑定 |
| `main()` | 顺序执行四模块 + 格式化报告输出 |

异常处理：`KeyboardInterrupt` 捕获 + 通用 `Exception` traceback 打印

## 十二、技术选型理由

| 技术 | 选型 | 理由 |
|------|------|------|
| SHA-256 | 哈希算法 | 区块链标准，256位输出，抗碰撞 |
| RSA-2048 | 非对称加密 | 成熟可靠，~112位安全强度 |
| OAEP | 加密填充 | 最安全的 RSA 填充，防选择密文攻击 |
| RSA-PSS | 签名方案 | 比 PKCS#1 v1.5 更安全，防签名伪造 |
| cryptography | 密码学库 | Python 生态最佳密码学库，维护活跃 |
