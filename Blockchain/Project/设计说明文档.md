# 现实世界资产代币化系统 (RWA) — 设计说明文档

---

## 一、系统概述

### 1.1 项目定位

本系统是一套基于以太坊智能合约的现实世界资产代币化（RWA）平台，将房地产、股权、债券、大宗商品等传统资产通过区块链技术数字化，实现资产的链上登记、合规发行、二级市场流通、自动化收益分配与全流程风险管理。

### 1.2 设计要求覆盖

系统完整覆盖《项目设计要求文档》4.1-4.4 全部模块：

| 设计模块 | 对应合约 | 合约数量 |
|----------|----------|:--------:|
| 4.2.1 资产登记层 | `AssetRegistry.sol` + `RWAToken.sol` | 2 |
| 4.2.2 合规发行系统 | `Compliance.sol` | 1 |
| 4.2.3 二级市场交易 | `Marketplace.sol` + `DutchAuction.sol` | 2 |
| 4.3.1 智能合约法律等效 | `LegalEnforcement.sol` | 1 |
| 4.3.2 动态估值系统 | `Valuation.sol` | 1 |
| 4.3.3 收益分配自动化 | `RevenueDistributor.sol` | 1 |
| 4.4 风险管理框架 | `RiskManager.sol` + `Insurance.sol` | 2 |
| **合计** | | **10** |

**合约统计**：10 个独立合约 | 22 个结构体 | 6 个枚举 | 67 个事件 | 125 个 public/external 函数

**合约依赖关系**：

- `Compliance`：独立部署，被 RWAToken、Marketplace、RevenueDistributor、RiskManager 引用
- RWAToken：需要 `setComplianceContract()` 设置合规引用，被 Marketplace、RevenueDistributor、DutchAuction 引用
- Marketplace：构造参数依赖 Compliance + RWAToken 地址
- RiskManager：构造参数依赖 Compliance 地址，运行时通过 `setInsuranceContract()` 关联 Insurance
- AssetRegistry、LegalEnforcement、Valuation、DutchAuction、Insurance：无依赖，可独立部署

### 3.1 AssetRegistry.sol — 资产登记层（4.2.1）

**功能**：多资产类型上链登记、SPV 风险隔离、信息披露管理

#### 数据结构

```solidity
enum AssetCategory { RealEstate, Equity, Bond, Commodity }

struct AssetRecord {
    uint256 assetId;
    string name;
    AssetCategory category;     // 房地产/股权/债券/大宗商品
    uint256 totalValue;         // 注册总估值
    uint256 tokenizedAmount;    // 已代币化数量
    address spvAddress;         // SPV 隔离地址
    address tokenAddress;       // 对应代币合约
    address issuer;             // 发行方
    uint256 registeredAt;       // 登记时间戳
    bool isActive;
    string legalDocHash;        // 法律文件 IPFS 哈希
}

struct Disclosure {
    uint256 disclosureId;
    uint256 assetId;
    string title;
    string contentHash;         // 内容 IPFS 哈希
    uint256 publishedAt;
    bool active;
}
```

#### 核心函数

| 函数 | 说明 |
|------|------|
| `registerAsset(name, category, totalValue, spvAddress, tokenAddress, legalDocHash) → uint256` | 登记新资产 |
| `updateTokenizedAmount(assetId, amount)` | 更新代币化进度 |
| `updateAssetValue(assetId, newValue)` | 更新资产估值 |
| `deactivateAsset(assetId)` | 停用资产 |
| `publishDisclosure(assetId, title, contentHash) → uint256` | 发布信息披露 |
| `getAssetDisclosures(assetId) → Disclosure[]` | 查询资产信息披露 |
| `getSPVAssets(spvAddress) → uint256[]` | 查询 SPV 下所有资产 |
| `getTokenizationRate(assetId) → uint256` | 代币化率（百分比） |

### 3.2 Compliance.sol — 合规引擎（4.2.2）

**功能**：KYC 认证 + AML 反洗钱 + 发行限制 + 转让限制 + 黑名单

#### 数据结构

```solidity
struct Investor {
    address wallet;
    string name;
    string kycId;
    uint256 verifiedAt;
    uint256 expiresAt;
    bool isAccredited;          // 合格投资者
    bool isBlacklisted;         // 黑名单
    uint256 maxInvestment;      // 最大投资额
}

struct TransferRestriction {
    uint256 minHoldingPeriod;   // 最短持有期（秒）
    uint256 maxTransferAmount;  // 单次最大转让额
    bool lockupActive;          // 锁定期开关
    uint256 lockupEnd;          // 锁定期结束时间
}

struct IssuanceLimit {
    uint256 maxSupply;              // 最大供应量
    uint256 maxInvestors;           // 最大投资者数
    uint256 minLockupPeriod;        // 最短锁定期
    uint256 maxInvestmentPerInvestor; // 单人最大投资额
    bool enabled;
}
```

#### 核心函数

| 类别 | 函数 | 说明 |
|------|------|------|
| KYC | `verifyInvestor(investor, name, kycId, isAccredited, maxInvestment, validityPeriod)` | 单地址认证 |
| KYC | `batchVerifyInvestors(investors[], names[], kycIds[], ...)` | 批量认证 |
| KYC | `revokeInvestor(investor, reason)` | 撤销认证 |
| KYC | `isVerified(address) → bool` | 查询认证状态 |
| 发行 | `setIssuanceLimit(maxSupply, maxInvestors, minLockupPeriod, maxInvestmentPerInvestor, enabled)` | 设置发行限制 |
| 发行 | `checkIssuance(currentSupply, currentInvestorCount, mintAmount) → (bool, string)` | 发行合规检查 |
| 转账 | `checkTransfer(from, to, amount) → (bool, string)` | 转账合规检查 |
| AML | `setAMLThreshold(threshold)` | 设置大额交易阈值 |
| AML | `setAMLRiskLevel(account, riskLevel)` | 风险等级（0-3） |
| AML | `recordTransaction(account, amount)` | 记录累计交易（仅管理员） |
| AML | `clearAMLFlag(account)` | 清除 AML 标记 |
| 黑名单 | `updateBlacklist(account, status)` | 单个黑名单管理 |
| 黑名单 | `batchUpdateBlacklist(accounts[], statuses[])` | 批量黑名单管理 |

**AML 机制**：追踪每个账户 `cumulativeTradeVolume`，超过 `amlThreshold`（默认 50000 ETH）自动标记 `amlFlagged`。支持四级风险等级（0=安全, 1=低风险, 2=中风险, 3=高风险）。

### 3.3 RWAToken.sol — 资产代币（4.1 / 4.2.1）

**功能**：ERC-20 代币 + 白名单机制 + 合规检查 + 暂停控制

#### 核心状态

```solidity
string public name;                // 代币名称
string public symbol;              // 代币符号
uint8 public decimals = 18;
uint256 public totalSupply;
string public assetType;           // 资产类型
uint256 public assetValue;         // 资产估值（美元）
address public spvAddress;         // SPV 地址
address public complianceContract; // 合规合约引用
mapping(address => bool) public whitelisted; // 白名单
bool public paused;                // 暂停标志
bool public complianceCheckEnabled; // 合规检查开关
```

#### 核心函数

| 函数 | 说明 |
|------|------|
| `mint(to, amount)` | 铸造代币（仅管理员） |
| `batchMint(recipients[], amounts[])` | 批量铸造 |
| `burn(amount)` | 销毁代币 |
| `transfer(to, amount) → bool` | 转账（含白名单 + 合规双重检查） |
| `updateWhitelist(account, status)` | 单个白名单管理 |
| `batchUpdateWhitelist(accounts[], statuses[])` | 批量白名单管理 |
| `updateAssetValue(newValue)` | 更新资产估值 |
| `getTokenPrice() → uint256` | 代币单价 = assetValue / totalSupply |
| `getTokenInfo() → (name, symbol, assetType, assetValue, totalSupply, tokenPrice, whitelistedCount)` | 完整代币信息 |
| `pause()` / `unpause()` | 紧急暂停/恢复 |
| `toggleComplianceCheck(bool)` | 合规检查开关 |

**转账流程**：

```
transfer(to, amount)
  ├── require(!paused, "contract is paused")
  ├── require(whitelisted[from] && whitelisted[to], "not whitelisted")
  ├── if complianceCheckEnabled → checkTransfer(from, to, amount)
  └── 标准 ERC-20 转账
```

### 3.4 Marketplace.sol — 交易市场（4.2.3）

**功能**：订单簿交易 + 做市商系统 + OTC 场外交易

#### 数据结构

```solidity
struct SellOrder {
    uint256 orderId;
    address seller;
    address tokenAddress;
    uint256 amount;             // 剩余数量
    uint256 totalAmount;        // 原始数量
    uint256 pricePerToken;      // 单价（wei）
    uint256 timestamp;
    bool active;
}

struct MarketQuote {
    address maker;
    address token;
    uint256 bidPrice;           // 买入价
    uint256 askPrice;           // 卖出价
    uint256 bidAmount;          // 买入量
    uint256 askAmount;          // 卖出量
    uint256 updatedAt;
    bool active;
}

enum OTCStatus { Pending, Accepted, Settled, Cancelled }

struct OTCTrade {
    uint256 tradeId;
    address partyA;             // 发起方
    address partyB;             // 对手方
    address token;
    uint256 amount;
    uint256 price;
    uint256 createdAt;
    uint256 settledAt;
    OTCStatus status;
}
```

#### 核心函数

| 交易模式 | 函数 | 说明 |
|----------|------|------|
| 订单簿 | `createSellOrder(token, amount, pricePerToken) → uint256` | 挂卖单 |
| 订单簿 | `buyTokens(orderId, amount)` | 购买（支持部分成交） |
| 订单簿 | `buyAllTokens(orderId)` | 全额购买 |
| 订单簿 | `cancelSellOrder(orderId)` | 取消订单 |
| 订单簿 | `getOrderTotal(orderId, amount) → (totalPrice, fee)` | 含手续费总价 |
| 订单簿 | `getActiveOrders() → uint256[]` | 活跃订单列表 |
| 做市商 | `registerMarketMaker(maker)` | 注册做市商 |
| 做市商 | `updateQuote(token, bidPrice, askPrice, bidAmount, askAmount)` | 更新双向报价 |
| 做市商 | `buyFromMaker(maker, token, amount)` | 以 ask 价买入 |
| 做市商 | `sellToMaker(maker, token, amount)` | 以 bid 价卖出 |
| OTC | `proposeOTC(partyB, token, amount, price) → uint256` | 发起交易意向 |
| OTC | `acceptOTC(tradeId)` | 接受意向（Pending → Accepted） |
| OTC | `settleOTC(tradeId)` | 结算（Accepted → Settled） |
| OTC | `cancelOTC(tradeId)` | 取消（→ Cancelled） |

**手续费**：`feeRate = 25`（0.25%），每笔成交自动扣除。

**安全防护**：`whenNotPaused` 修饰符 + `nonReentrant` 修饰符覆盖全部资金操作函数。

    uint256 endAt;

    uint256 settledAt;

| `cancelAuction(auctionId)` | 取消拍卖并退回代币 |

### 3.8 Valuation.sol — 动态估值系统（4.3.2）

**功能**：四种估值方法 + 加权综合估值

#### 数据结构

```solidity
enum Method { DiscountedCashFlow, ComparableCompany, Cost, Market }

struct ValuationRecord {
    uint256 valuationId;
    uint256 assetId;
    Method method;              // 估值方法
    uint256 estimatedValue;     // 估计价值
    uint256 timestamp;
    string paramsHash;          // 参数 IPFS 哈希
    address appraiser;          // 评估人
    bool verified;
}
```

#### 核心函数

| 方法 | 函数 | 计算公式 |
|------|------|----------|
| DCF | `valuationByDCF(assetId, cashFlows[], discountRate, paramsHash) → uint256` | Σ(cashFlow / (1 + r)^n) |
| 可比公司 | `valuationByComparable(assetId, comparableValue, adjustmentRate, paramsHash) → uint256` | comparableValue × (1 + adjustmentRate/10000) |
| 成本法 | `valuationByCost(assetId, replacementCost, depreciationRate, paramsHash) → uint256` | replacementCost × (1 - depreciationRate/10000) |
| 市场法 | `valuationByMarket(assetId, recentTradePrice, marketAdjustFactor, paramsHash) → uint256` | recentTradePrice × marketAdjustFactor / 10000 |
| 综合 | `computeCompositeValue(assetId, valuationIds[], weights[]) → uint256` | Σ(value_i × weight_i) / Σ(weight_i) |
| 查询 | `getLatestCompositeValue(assetId) → uint256` | 最新加权综合估值 |
| 查询 | `getAssetValuations(assetId) → uint256[]` | 资产全部估值 ID |
| 验证 | `verifyValuation(valuationId)` | 标记估值已验证 |

### 3.9 RiskManager.sol — 风险管理（4.4）

**功能**：抵押仓位管理 + 清算机制 + 压力测试 + 保险绑定

#### 数据结构

```solidity
struct Position {
    address owner;
    address collateralToken;
    address borrowToken;
    uint256 collateralAmount;
    uint256 borrowAmount;
    uint256 createdAt;
    bool isActive;
}

struct RiskParams {
    uint256 minCollateralRatio;    // 最低抵押率（默认 150%）
    uint256 liquidationThreshold;  // 清算阈值（默认 120%）
    uint256 liquidationPenalty;    // 清算罚金（默认 10%）
    uint256 maxLeverage;           // 最大杠杆（默认 3x）
}

struct StressScenario {
    uint256 scenarioId;
    string name;
    uint256 priceDropPercent;      // 跌幅（基点）
    uint256 liquidationRatio;      // 预期清算比例
    uint256 testedAt;
    uint256 affectedPositions;     // 受影响仓位数量
    bool passed;
    string reportHash;
}
```

#### 核心函数

| 类别 | 函数 | 说明 |
|------|------|------|
| 仓位 | `createPosition(collateralToken, borrowToken, collateralAmount, borrowAmount) → uint256` | 创建抵押仓位 |
| 仓位 | `addCollateral(positionId, amount)` | 追加抵押品 |
| 仓位 | `repayDebt(positionId, amount)` | 归还借款 |
| 仓位 | `closePosition(positionId)` | 平仓 |
| 清算 | `liquidate(positionId)` | 清算不健康仓位（任何人可触发） |
| 清算 | `liquidateWithInsurance(positionId)` | 清算 + 同步触发保险理赔 |
| 清算 | `isLiquidatable(positionId) → bool` | 检查是否满足清算条件 |
| 监控 | `getCollateralRatio(positionId) → uint256` | 抵押率 = collateralAmount / borrowAmount |
| 监控 | `isPositionHealthy(positionId) → bool` | 健康判断（抵押率 > 清算阈值） |
| 监控 | `detectUnhealthyPositions() → uint256[]` | 检测所有不健康仓位 |
| 压力测试 | `runStressTest(name, priceDropPercent, reportHash) → uint256` | 模拟价格下跌场景 |
| 保险 | `bindInsurance(positionId, policyId)` | 仓位绑定保单 |
| 保险 | `getPositionInsurance(positionId) → (insured, policyId)` | 查询绑定状态 |
| 参数 | `updateRiskParams(minRatio, liqThreshold, liqPenalty, maxLeverage)` | 更新风险参数 |
| 参数 | `setSafetyThreshold(threshold)` | 设置系统安全阈值 |
| 统计 | `getSystemRiskStats() → (...)` | 系统风险统计 |

### 3.10 Insurance.sol — 保险集成（4.4）

**功能**：保单发行与管理 + 理赔申请与审批

#### 数据结构

```solidity
enum PolicyStatus { Active, Claimed, Expired, Cancelled }
enum ClaimStatus { Pending, Approved, Rejected, Paid }

struct Policy {
    uint256 policyId;
    uint256 assetId;
    address insured;            // 被保险人
    address insurer;            // 承保人
    uint256 coverageAmount;     // 保额
    uint256 premium;            // 保费
    uint256 startAt;
    uint256 endAt;              // 到期时间
    PolicyStatus status;
    string termsHash;           // 条款 IPFS 哈希
}

struct Claim {
    uint256 claimId;
    uint256 policyId;
    address claimant;
    uint256 claimAmount;        // 理赔金额
    string reasonHash;          // 理赔原因 IPFS 哈希
    uint256 filedAt;
    uint256 processedAt;
    ClaimStatus status;
}
```

#### 核心函数

| 函数 | 说明 |
|------|------|
| `issuePolicy(assetId, insured, coverageAmount, premium, duration, termsHash) → uint256` | 发行保单 |
| `cancelPolicy(policyId)` | 退保 |
| `fileClaim(policyId, claimAmount, reasonHash) → uint256` | 发起理赔（→ Pending） |
| `approveClaim(claimId)` payable | 批准理赔并支付（→ Paid，仅管理员） |
| `rejectClaim(claimId)` | 拒绝理赔（→ Rejected） |
| `isPolicyActive(policyId) → bool` | 保单有效性检查 |
| `getInsuranceStats() → (policyCount, claimCount, totalCoverage, totalPremium, totalClaimPaid)` | 保险统计 |

## 四、前端设计

### 4.1 页面结构

前端采用侧边栏导航 + 主内容区布局，共 10 个子页面：

```
侧边栏导航：
  ⊞ 总览         → 资产信息 + 账户状态 + 快速操作（铸造/白名单/KYC）
  ↯ 交易市场     → 子导航：订单簿 | 做市商 | OTC 交易
  $ 收益分配     → 收益存入 + 分配创建 + 领取 + 再投资
  ⬡ 资产登记     → SPV 隔离资产列表 + 信息披露
  ∿ 动态估值     → 综合估值 + 四种方法估值记录
  ⓘ 荷兰拍卖     → 拍卖列表 + 实时价格 + 竞价
  ◊ 保险集成     → 保单列表 + 理赔申请
  § 法律等效     → 条款 / 争议 / 事件 / 报告
  ⚠ 风险管理     → 仓位管理 + 压力测试 + 保险绑定
  ⛔ AML 合规    → AML 监测 + 发行限制 + 风险等级管理
```

### 4.2 技术实现

| 组件 | 实现方式 |
|------|----------|
| RPC 连接 | 通过 `server.js` 的 `/rpc` 代理转发，POST 转发到 Hardhat Node :8545 |
| 钱包签名 | 内置 Hardhat Account #0 私钥创建 `ethers.Wallet`，免 MetaMask 交互 |
| 合约实例 | 10 个 `ethers.Contract`，均以 signer 创建（支持读 + 写操作） |
| 数据刷新 | 页面切换时调用对应 refresh 函数，并行获取链上数据 |
| 弹窗系统 | 通用 `showModal({type, title, message, ...})` 支持 alert / confirm / input 模式 |
| Toast 通知 | 操作结果以 toast 横幅提示，5 秒自动消失 |
| 响应式布局 | 桌面端双栏 → 平板单栏 → 移动端堆叠，断点 768px / 480px |
| 子导航切换 | 交易市场内通过 JavaScript 动态切换订单簿 / 做市商 / OTC 三个子视图 |

## 五、部署架构

### 5.1 部署流程

```
rwa.sh
  │
  ├── 1. npm install                   — 安装依赖
  ├── 2. npx hardhat compile           — 编译 10 个合约
  ├── 3. npx hardhat node &            — 后台启动 Hardhat Node
  ├── 4. npx hardhat run scripts/deploy.js --network localhost
  │       └── 10 个合约按依赖顺序部署
  │           Compliance → RWAToken → Marketplace → RevenueDistributor
  │           → RiskManager → AssetRegistry → LegalEnforcement
  │           → Valuation → DutchAuction → Insurance
  ├── 5. npx hardhat run scripts/init-data.js --network localhost
  │       └── 补充演示数据（卖单/做市商/拍卖/OTC/收益/保险/估值/仓位/压力测试）
  ├── 6. node scripts/sync-addresses.js — 同步合约地址到 frontend/app.js
  └── 7. node server.js                — 启动前端服务（:8080）
```

## 六、安全设计

### 6.1 访问控制

| 角色 | 权限范围 | 涉及合约 |
|------|----------|----------|
| 管理员 (admin) | 铸造代币、白名单管理、KYC 认证、估值、发行保单、参数设置 | 全部 10 个合约 |
| 仲裁员 (arbitrator) | 争议裁决 | LegalEnforcement |
| 认证用户 | 转账、交易、领取收益、创建拍卖 | RWAToken / Marketplace / Revenue / DutchAuction |
| 未认证用户 | 仅查看（public view 函数） | 全部 |

### 6.2 安全机制汇总

| 机制 | 实现 | 覆盖函数数 |
|------|------|:----------:|
| 重入防护 | 自定义 `nonReentrant` 修饰符（`_reentrancyStatus` 状态位） | 16 |
| 暂停控制 | `whenNotPaused` 修饰符 | RWAToken(2) + Marketplace(8) |
| 白名单校验 | `onlyWhitelisted` 修饰符 + 转账前检查 | RWAToken transfer |
| 合规检查 | 转账/交易前调用 `checkTransfer(from, to, amount)` | RWAToken + Marketplace |
| 黑名单 | 封禁违规地址 | Compliance |
| 路径穿越防护 | 服务端 `filePath.startsWith(FRONTEND_DIR)` 检查 | server.js |
| 整数溢出防护 | Solidity 0.8.28 内置检查 | 全部 10 合约 |
| 超额支付退款 | 买方支付超额的 ETH 自动退还 | Marketplace |
